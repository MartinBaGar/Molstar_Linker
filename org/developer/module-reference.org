#+TITLE: Module Reference
#+WEIGHT: 30

All source files are in =src/= and written in TypeScript. Each entry-point module is compiled by esbuild into a single self-contained =.js= file in =dist/=. Shared modules (=types=, =config=, =permissions=, =native-builder=) are bundled into the entry points that import them.

* Shared Modules

** =src/types.ts=

Defines TypeScript interfaces and the inter-context message protocol.

Key exports:
- =RepType=: Union of valid Mol* representation strings (="cartoon"=, ="ball_and_stick"=, etc.).
- =RuleRepType=: Extends =RepType= with ="highlight"= for custom rules.
- =TargetDefinition=: Schema for molecular components (proteins, ligands, etc.).
- =CustomRule=: Schema for user-defined visual rules.
- =ExtensionSettings=: The master settings object stored in =chrome.storage.sync=.
- =Preset=: Schema for named templates.
- Message types: =OpenViewerMessage=, =InitMolstarMessage=, =SandboxReadyMessage=.

** =src/config.ts=

Central configuration for the extension.

Key exports:
- =AppConfig.RepSchema=: Maps representation keys to labels and configurable parameters.
- =AppConfig.targets=: Array of =TargetDefinition= objects (protein, nucleic, ligand, etc.).
- =AppConfig.presets=: Built-in templates (e.g., "Dark Mode").
- =AppConfig.getDefaults()=: Generates a baseline =ExtensionSettings= object.

** =src/permissions.ts=

Manages dynamic host permissions for custom domains.

Key exports:
- =PermissionsManager.cleanDomain(url)=: Extracts the hostname from a URL.
- =PermissionsManager.requestAndRegister(url)=: Requests host permission, registers the content script, and persists the domain.
- =PermissionsManager.revokeAndUnregister(url)=: Revokes permission and removes the domain.

** =src/native-builder.ts=

Core logic for building Mol* scenes natively.

Key features:
- =buildNativeScene()=: Entry point for rendering a structure with:
  - Global target configurations (proteins, ligands, etc.).
  - Custom rules (highlighting, labels, tooltips).
  - Camera settings.
- =applyRepresentation()=: Applies visual styles to a molecular component.
- =applyCustomRuleRepresentation()=: Applies styles for custom rules.
- =apply3DLabel()=: Adds floating 3D labels to the scene.
- Tooltip management via Mol*'s =lociLabels= provider.

* Entry-Point Modules

** =src/background.ts= → =background.js=

Background script (service worker in Chrome, event page in Firefox).

Responsibilities:
- Routes =open_viewer= messages from content scripts to the viewer.
- Validates URLs (HTTPS-only) and formats before opening =viewer.html=.
- Manages the right-click context menu (*"Open in Mol* Workspace"*).
- Dynamically registers content scripts for newly authorized domains.

** =src/content.ts= → =content.js=

Content script injected into host webpages.

Key functions:
- =getStructureInfo(href, linkElement)=: Detects structure files using a 3-ring scanner (URL, HTML attributes, parent text).
  - Transforms GitHub/GitLab UI URLs to raw file URLs.
  - Uses GitLab API for self-hosted instances.
- =makeBadge(rawUrl, formatStr, originalHref)=: Creates interactive *Mol** badges.
- =injectMolstarLinker()=: Main scan pass; skips processed links and avoids duplicates.
- =MutationObserver=: Debounced scanner for dynamic content (e.g., SPAs).

** =src/viewer.ts= → =viewer.js=

Privileged shell with full extension API access.

Key functions:
- =isSafeUrl(url)=: Validates HTTPS and blocks private IP ranges.
- =bootWorkspace(rawUrl, safeFormat)=: Fetches the file, enforces size limits, and spawns the sandbox.
- =spawnIframe(dataUri, format, rawUrl)=: Initializes the sandbox iframe and sends settings.

** =src/sandbox.ts= → =sandbox.js=

Sandboxed iframe for Mol* rendering.

Flow:
1. Signals readiness with =SANDBOX_READY=.
2. Validates =INIT_MOLSTAR= messages.
3. Converts base64 data URIs to blob URLs.
4. Initializes the Mol* viewer and loads the structure.
5. Delegates rendering to =NativeBuilder=.

** =src/popup.ts= → =popup.js=

Toolbar popup UI.

Features:
- Applies presets to the current tab.
- Opens the Options page.
- Detects unauthorized domains and prompts for authorization.

** =src/options.ts= → =options.js=

Advanced settings UI.

Key functions:
- =buildUI()=: Generates target configuration cards.
- =addCustomRuleCard(ruleData?)=: Creates custom rule forms with simple/expert mode.
- =extractCurrentSettings()=: Reads UI state into =ExtensionSettings=.
- =injectSettingsIntoUI(settings)=: Populates the UI from settings.
- Template and domain management.
- JSON import/export with validation.
