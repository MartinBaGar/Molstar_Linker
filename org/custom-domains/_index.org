* Authorizing Custom Domains

Mol* Linker works out-of-the-box on GitHub, GitLab, RCSB, and AlphaFold. To use it on other domains (e.g., self-hosted GitLab, university servers, or research databases), you must authorize the domain.

* Why Authorization is Needed

Browsers enforce strict security policies to prevent extensions from accessing arbitrary websites. Authorizing a domain grants Mol* Linker permission to:
- Scan the page for structure files.
- Inject *Mol** badges.
- Fetch files securely.

* How to Add a Custom Domain

**Method 1: From the Extension Popup**

1. Navigate to the website you want to authorize (e.g., =https://gitlab.my-university.edu=).
2. Click the *Mol* Linker* extension icon in your browser toolbar.
3. If the domain is not authorized, you will see a prompt to *Authorize in Studio*.
4. Click the button to open the Options page.
5. The domain will be pre-filled in the *Manual Domain* input. Click *Add* to authorize it.

**Method 2: From the Options Page**

1. Open the Mol* Linker Options page:
   - Click the extension icon → *Open Advanced Options*.
   - Or right-click the extension icon → *Options*.
2. Scroll to the *Custom Domains* section.
3. Enter the domain (e.g., =gitlab.my-university.edu=) in the *Manual Domain* input.
4. Click *Add*.

**Method 3: Right-Click Fallback**

1. On a page with structure files, *right-click* a download link.
2. Select *"Open in Mol* Workspace"* from the context menu.
3. If the domain is not authorized, you will be prompted to authorize it.
4. Click *Authorize* to grant access.

* Managing Authorized Domains

1. Open the Mol* Linker Options page.
2. Scroll to the *Custom Domains* section.
3. You will see a list of all authorized domains.
4. Click *Remove* next to any domain to revoke access.

* How It Works

When you authorize a domain:
1. The extension requests permission to access the domain from your browser.
2. If granted, the extension dynamically registers a content script for the domain.
3. The content script scans the page for structure files and injects *Mol** badges.

This process is secure and does not require server-side changes (e.g., CORS headers).

* Troubleshooting

- *Badges don't appear*: Ensure the domain is authorized and the page has been refreshed.
- *Files fail to load*: Check that the domain is correctly spelled in the *Custom Domains* list.
- *Permission denied*: Some browsers may require you to manually enable permissions in the extension management page (=chrome://extensions/= or =about:addons=).
