Once the extension is installed, no configuration is needed to get started.

* See Your First Structure

1. Navigate to any GitHub or GitLab repository that contains a =.pdb=, =.cif=, or other [[../../#supported-formats][supported file]].
2. Look for a green *Mol** badge next to file names in the file browser or next to links in README files.
3. Click the badge — the structure opens in the Mol* viewer in a new tab, rendered natively with your current settings.

* Change the Active Preset

Click the extension icon in your browser toolbar to open the popup:

1. Select a preset from the *Preset* dropdown (e.g., "Dark Mode Canvas" or "Protein Surface + Spacefill Ligands").
2. Click *Apply Preset*.
3. The current page reloads, and new badges will use the selected preset.

* Open the Advanced Options Page

Click *Open Advanced Options* in the popup or right-click the extension icon and choose *Options*. The Options page lets you:
- Configure representations, colors, and transparency for each molecular component.
- Define custom rules for highlighting specific chains, residues, or atoms.
- Save and manage presets.
- Authorize custom domains.

See [[../configuration][Configuration]] for details.

* Working with Custom Domains

Mol* Linker works out-of-the-box on GitHub, GitLab, RCSB, and AlphaFold. For other domains (e.g., self-hosted GitLab, university servers):

1. Navigate to a page with a structure file.
2. Click the extension icon in the toolbar.
3. If the domain is not authorized, you will see a prompt to *Authorize in Studio*.
4. Click the button to open the Options page and grant access.

* Right-Click Fallback

If the badge does not appear automatically:
1. **Right-click** the download link or button.
2. Select **"Open in Mol* Workspace"** from the context menu.
3. If the format is unknown, you will be prompted to select it from a dropdown before launching the viewer.
